# Mechanical Systems and Structures Design Portfolio

This repository contains a complete static portfolio website for a mechanical engineering student specializing in **Mechanical Systems and Structures Design**.

The site is built with:

- `HTML`
- `CSS`
- `JavaScript`

No framework is required, so it works well with **GitHub Pages**.

## Files

Place these files in the **root of your repository**:

- `index.html`
- `style.css`
- `script.js`
- `README.md`

Your repository should look like this:

```text
mechanical-systems-and-structures-design/
|-- index.html
|-- style.css
|-- script.js
`-- README.md
```

## What to Customize Before Publishing

Update the placeholders in `index.html` with your own information:

- Featured project titles and descriptions
- Software and technical skills
- Email address
- GitHub profile link
- LinkedIn profile link

You can also later replace the project placeholder visual blocks with real project screenshots, CAD renders, simulation plots, or workshop photos.

## How to Publish with GitHub Pages

1. Create or open the GitHub repository named `mechanical-systems-and-structures-design`.
2. Upload these four files to the repository root.
3. Commit and push the files to your default branch (`main` in most cases).
4. On GitHub, open the repository.
5. Go to **Settings**.
6. In the left sidebar, click **Pages**.
7. Under **Build and deployment**, choose:
   - **Source:** `Deploy from a branch`
   - **Branch:** `main`
   - **Folder:** `/ (root)`
8. Click **Save**.
9. Wait a minute or two for GitHub to publish the site.
10. Open the published URL:

```text
https://YOUR-GITHUB-USERNAME.github.io/mechanical-systems-and-structures-design/
```

If your default branch is `master` instead of `main`, select `master` in the GitHub Pages settings.

## Local Preview

To preview the website locally, just open `index.html` in your browser.

## Notes

- The website is fully static and compatible with GitHub Pages.
- Navigation uses anchor links and smooth scrolling.
- The mobile menu, project detail toggles, section reveal animation, and back-to-top button are handled by `script.js`.
- Styling is centralized in `style.css` to keep edits simple later.
